合成データで安全に研究データを共有する

code-to-data / TRE 型ワークフローのデモ — 生データを外部に出さず、合成データで開発した分析コードを オーナーが生データへ適用する

  1. 1合成データ公開
  2. 2分析者が開発・提出
  3. 3オーナーが承認・生データへ適用

① オーナー：合成データを公開する

手元の生データ(機微情報)はサーバー外に出しません。統計的性質を模した 合成データだけを分析者に公開します。

🔒 非公開

生データ (raw)

実際の患者の PSA 時系列・投薬記録。TRE 内に隔離され、分析者からは 参照できません。

site/data/raw.json
🔓 公開済み

合成データ (synthetic)

別シード・分布シフトで生成。統計的に類似するが個票は別物。分析者は これを使ってコードを開発します。

site/data/synthetic.json

② 分析者：合成データで分析を開発する

公開された合成データに対して分析コードを実行します (本番では htmx が FastAPI のエンドポイントを叩き、サーバーが scikit-learn で 計算します)。

上のボタンから分析を選ぶと、合成データでの結果が ここに読み込まれます。

提出済み: → 審査タブへ

③ オーナー：提出物を審査し、生データに適用する

分析者が提出したのは合成データでの結果です。コードを確認し、 問題なければ同じコードを生データに適用して本物のレポートを 作成します。

提出物：合成データの結果

承認後：生データの結果

「承認して生データに適用」を押すと、同一コードを 生データで実行した結果がここに表示されます。

✅ 同じ分析コードが両データで動作しました。結果は近いが完全には 一致しません(個票が異なるため)。だからこそ、合成データで安全に 開発し、最後に生データへ適用するステップに価値があります。

静的デモ (GitHub Pages) — htmx + Alpine.js + Chart.js。データと分析は Python (pandas / scikit-learn) で事前計算。仕様書
